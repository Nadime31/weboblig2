STRUKTUR: FLEXBOX / GRID

Struktur:

Normalize.css: Gi deg selv best mulig utgangspunkt for å bygge struktur

Planlegg struktur etter en wireframe/skisse.

"Old school” CSS :

Wrappere brukes for å sørge for rad-inndeling. Boxer brukes for å sørge for flyt inne wrappere.Størrelser er i % for responsivitet / grid

The “Flex”ible way:CSS Flexbox:

Flex-container for å sørge for rad- inndeling.Flex-items for å sørge for flyt inne wrappere.Størrelser i % for responsivitet / grid.

CSS FlexBox container

Display: flex; forteller containeren hvordan items innenfor skal vises.
Justify-content: space-between;forteller containeren at plass mellom elementer skal utjevnes på en rad.
Flex-wrap: wrap;forteller containeren at elementer skal wrappes hvis de overstiger en rad.

CSS FlexBox item

Flex: 1; forteller item at den skal automatisk oppta plassen til ett element. Hvis det er 5 elementer, vil den oppta 1/5 av den tilgjengelige bredden. Item kan få angitt bredde på “vanlig vis” med prosentstørrelser. Flex-containeren vil da bryte items over flere rader (hvis containeren har flex-wrap: wrap;)
Order: 2; forteller item hvilken rekkefølge dette itemet skal legge seg i.

The futuristic way:CSS Grid:

Grid-container for å sørge for områder i en grid. Grid-seksjoner plasseres i grid- områdene.

CSS rammeverk

  • Ferdiglaget CSS for å strukturere HTML
  • De fleste bruker “old school” CSS
  • Noen har begynt å bruke Flexbox
  • Fordeler: Sparer mange linjer kode.
  • Kommer fort i gang med teknisk design.
  • Ulemper: Mye ekstra kode. Må læres.

Foreslått best løsning:

Bruk CSS Grid, og design etter responsive prinsipper. For nettlesere som ikke støtter CSS Grid, gi dem mobilversjonen av siden. Denne vil alltid vises i lesbart format.
